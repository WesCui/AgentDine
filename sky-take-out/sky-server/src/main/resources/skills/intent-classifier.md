# 意图分类规则

## 分类逻辑
根据用户输入的关键词和语义，将请求路由到对应的专业Agent。

## 意图 → Agent 映射

### food-recommendation（点餐推荐）
关键词：点餐、推荐、想吃、点什么、有什么菜、来一份、推荐菜、饿了
语义：用户想浏览菜单、获取推荐、查询菜品信息

### order-management（订单管理）
关键词：订单、下单、查单、到哪了、配送、催单、取消、退款、支付
语义：用户想查询/操作订单状态

### customer-service（客服工单）
关键词：投诉、质量问题、不好吃、坏了、异物、态度差、超时、补偿
语义：用户有不满需要解决或投诉

### business-analytics（经营分析）
关键词：经营、销售、报表、数据、分析、统计、营业额、排行
语义：商家查询经营数据

### kitchen-display（后厨出餐）
关键词：后厨、出餐、制作、厨房、备餐、售罄
语义：后厨管理相关

## 默认规则
如果无法匹配任何意图，由 orchestrator 直接回答通用问题。
